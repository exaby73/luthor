## Single Values

`validateValue` returns `SingleValidationResult<T>`.

```dart
final result = l.string().email().required().validateValue('bad');

switch (result) {
  case SingleValidationSuccess(data: final email):
    print(email);
  case SingleValidationError(errors: final errors):
    print(errors); // [value must be a valid email address]
}
```

`isValid` is available on both result variants:

```dart
if (!result.isValid) {
  // inspect the error variant when you need messages
}
```

## Schemas

`validateSchema` returns `SchemaValidationResult<T>`.

```dart
final userSchema = l.schema({
  'email': l.string().email().required(),
  'profile': l.schema({
    'displayName': l.string().min(2).required(),
  }).required(),
});

final result = userSchema.validateSchema({
  'email': 'bad',
  'profile': {'displayName': 'x'},
});

switch (result) {
  case SchemaValidationSuccess(data: final data):
    print(data);
  case SchemaValidationError(errors: final errors):
    print(errors);
    print(result.getError('email'));
    print(result.getError('profile.displayName'));
}
```

Schema errors are maps. Nested schemas stay nested, and `getError('a.b')` returns the first message at that path.

## Typed Data

Pass `fromJson` when valid input should be converted to a model.

```dart
class User {
  final String email;

  User({required this.email});

  factory User.fromJson(Map<String, Object?> json) {
    return User(email: json['email']! as String);
  }
}

final result = l
    .schema({'email': l.string().email().required()})
    .validateSchema<User>(
      {'email': 'dev@example.com'},
      fromJson: User.fromJson,
    );
```

When schema validation fails and `fromJson` is provided, `SchemaValidationError.data` is `null` because conversion did not run.

## Generated Error Keys

With code generation, `ErrorKeys` constants remove stringly typed error lookups:

```dart
final emailError = result.getError(UserErrorKeys.email);
final cityError = result.getError(UserErrorKeys.address.city);
```
